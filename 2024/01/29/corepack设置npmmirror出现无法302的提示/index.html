corepack设置npmmirror出现无法302的提示

发表于 2024-01-29 分类于 corepack ， pnpm ， yarn ， nodejs

分析

corepack设置npmmirror出现无法302的提示

接上篇对corepack配置npmmirror为https://registry.npmmirror.com后,对pnpm进行设备更新时corepack use pnpm@latest 出现如下错误

直接在浏览器点开这个链接https://registry.npmmirror.com/pnpm/-/pnpm-8.15.0.tgz,会发现云端返回302后跳转到了cdn链接[https://cdn.npmmirror.com/packages/pnpm/8.15.0/pnpm-8.15.0.tgz]

但corepack内部将大于300的code全部定义为错误

还是老样子去github上看看是否已有类似的issue和PR，经过一番搜索，找到如下的讨论

add http redirect support #306

发现在#302已经解决，找到302
install from npmmirror.com failed (302 redirect) #302

在23年12月29日，有人已提交PR修复feat: Support redirections #341

我们来看看修复类容(PR 文件变更)(https://github.com/nodejs/corepack/pull/341/files)

变更也很简单，当code符合定义时，讲取出header.location进行第二次访问

结论

那我们要做的就很简单，升级到v0.24.0及以上。简单的升级办法是下载release里面版本符合的包，替换整个C:\Users\xxxx\AppData\Roaming\nvm\v20.11.0\node_modules\corepack目录即可
